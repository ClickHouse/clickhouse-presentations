Vector Similarity Search в ClickHouse

VSS — поиск похожих объектов в неструктурированных данных:

— Поиск похожих объектов по их embedding-ам.

— С помощью индексов в многомерных пространствах.

— В данных, превышающих объём памяти и одной машины.

— Индексы динамически обновляемые.

Актуальность, востребованность

Направление Vector Similarity Search развивается прямо сейчас
и совмещает в себе два современных направления инженерной мысли:
машинное обучение и обработка больших данных.

Примеры:

Анонс Google, December 13, 2021:
"Find anything blazingly fast with Google's vector search technology"
— закрытая технология для клиентов Облака.

Новые компании-стартапы вокруг этой технологии: Сohere.ai

Что будем делать

1. Подготовка датасетов:

— картинки из интернета (400 млн. картинок);

— комментарии в соц сетях (дамп Reddit, Hacker News);

— файлы с исходным кодом (10 млн. репозиториев);

— HTML страницы (6 млн сайтов).

Что будем делать

2. Изучение алгоритмов и их реализаций:

— hnswlib (Hierarchical Navigable Small World Graphs);
— redis-hnsw (реализация в Redis);
— nmslib (Non-Metric Space Library);
— Faiss;
— ScaNN;

Что будем делать

3. Baseline — проверка brute-force поиска:

— поиск полным перебором с вычислением Lp метрики налету.

Что будем делать

4. Выбор структуры данных для индекса:

— монолитный индекс, целиком загружаемый в память;

— разреженный индекс со статистиками по гранулам,
  варианты во внешней памяти;

— эксперименты по построению locality sensitive хэшей
  с помощью grid в VP-tree и space-filling curves.

Что будем делать

5. Реализация структур данных в ClickHouse.

--- здесь всё самое интересное ---

Что будем делать

6. Проведение экспериментов и сравнительное тестирование:

— потребление памяти;
— производительность поиска;
— скорость загрузки индекса;
— точность результатов.

Оптимизация производительности.

Разработка тестов и документации.
