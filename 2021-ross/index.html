History

2009 — report builder in Metrica — first ClickHouse prototype.

2010..2011 — development of ClickHouse as a side project with interruptions.

2012 — started using ClickHouse in production
for log processing.

2014 — Metrica 2.0 service fully based on ClickHouse.

2014..2015 — ClickHouse adoption within Yandex.

2016 — released ClickHouse as open-source.

History

2016..2017 — first external users
— enthusiasts with similar use cases.

2018 — ClickHouse launch in Yandex Cloud.

2018..2019 — active promotion, meetups in Europe, Asia and the USA.

2020 — continued growth due to WFH
and migration to ClickHouse in large companies.

Ways to Engage the Community

Low entry barrier:
— simplicity and clarity of code, presence of design docs;
— well-documented code — like a textbook.

Not only code is open, but also processes:
— open-source product development can be a reality show!

Core developers use the same tools as external contributors:
— send pull requests and merge them according to the same rules.

Discussions and plans are accessible externally:
— a contributor can choose a large task to implement,
and discuss ways to implement it.

Why?

Principles of successful open-source:

— solve a demanded problem;
— do it better than others*.

Goal:

ClickHouse should be the default choice
as an open-source analytical DBMS

— the only correct, obvious solution
for the corresponding range of tasks;

— and everyone should know about it :)

* — this condition is neither necessary nor sufficient;
